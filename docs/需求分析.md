# 情感本DM演绎设计助手 - 需求分析

## 1. 项目背景

### 1.1 什么是"剧本杀改剧本演绎"
剧本杀的"改剧本演绎"是指主持人（DM）或店家不照本宣科，而是根据玩家类型、现场氛围以及自身演绎能力，对剧本原有流程、台词或环节进行"二次创作"。这种操作的本质是"因地制宜"，目的是弥补剧本本身的瑕疵或硬件不足，让玩家获得比单纯阅读文字更极致的游戏体验。

### 1.2 核心痛点
- 优秀DM的演绎设计能力难以快速复制
- 情感本的心建和羁绊环节设计耗时费力
- 缺乏系统化的演绎方案沉淀和复用机制

## 2. 产品定位

### 2.1 目标用户
**剧本杀DM/店家**——开本前的准备阶段使用

### 2.2 核心价值
帮助DM快速设计**情感本**的一对一**仪式动作式**演绎环节，包括：
- 心建环节优化
- 特殊仪式设计（拜堂、羁绊仪式等）
- 玩家羁绊增强

### 2.3 使用场景
DM上传剧本 → 选择环节 → 选择演绎类型 → AI生成完整演绎手册 → DM预习后现场执行

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 剧本输入
- 支持文本粘贴输入
- 支持文件上传（PDF、Word、TXT）
- 剧本内容本地存储，支持历史查看

#### 3.1.2 剧本解析
- 自动解析剧本结构
- 识别角色关系（CP、亲情线等）
- 提取可改编的环节列表

#### 3.1.3 演绎设计
三级选择维度：
1. **情感类型**：爱情 / 亲情 / 友情 / 家国
2. **环节类型**：心建 / 羁绊仪式 / 情感爆发
3. **具体场景**：拜堂 / 对拜 / 递信物 / 回忆杀 等

#### 3.1.4 输出生成
生成**完整演绎手册**，包含：
- **动作指令**：具体动作描述（如"请玩家站起来，面向东方"）
- **DM台词**：完整台词文本 + 语气提示
- **节奏提示**：停顿点、推进时机、音乐配合建议
- **道具清单**：需要准备的物品列表

### 3.2 扩展功能（后续版本）

#### 3.2.1 双轨对比模式
DM可提供自己的改编想法，AI同时生成：
- AI自动方案
- DM想法的完善版本
- 两者的对比分析

#### 3.2.2 个性化定制
根据角色特点为不同玩家生成个性化演绎版本

## 4. 非功能需求

### 4.1 性能要求
- 剧本解析：文本<5000字时，解析时间<3秒
- 演绎生成：单个环节生成时间<10秒

### 4.2 可用性要求
- 简洁直观的操作流程
- 生成内容支持复制和导出
- 支持历史记录查看

### 4.3 安全要求
- 剧本内容本地存储，不上传云端
- 用户数据隔离

## 5. 用户故事

### 5.1 主要用户故事

**作为**一个情感本DM，
**我希望**上传剧本后能快速获得专业的演绎设计方案，
**从而**节省备课时间，提升开本质量。

**作为**一个新手DM，
**我希望**有系统的演绎设计指导，
**从而**快速掌握情感本的心建技巧。

**作为**一个资深DM，
**我希望**AI能提供演绎灵感，
**从而**在我的创意基础上进行完善和创新。

## 6. 术语表

| 术语 | 解释 |
|------|------|
| 情感本 | 以情感体验为核心的剧本杀类型 |
| 心建 | 开本前DM与玩家建立情感连接的准备环节 |
| DM | Dungeon Master，剧本杀主持人 |
| 羁绊 | 玩家之间或玩家与角色之间的情感联系 |
| CP | Character Pair，剧本中的情侣配对 |
| 开本 | 开始进行一场剧本杀游戏 |
